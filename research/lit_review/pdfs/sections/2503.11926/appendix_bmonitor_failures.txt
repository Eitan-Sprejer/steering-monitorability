Appendix B
Monitor Failures

We saw in Sections
2 and
3 that our monitors do not have perfect recall. We note that there is a lot of room for improvement with our monitor; in fact, the monitoring prompt (shown in Appendix
C
) was the first we experimented with. We furthermore did not train our models specifically for monitoring, but rather used existing chat models zero-shot. We show examples where our monitor failed to catch an instance of the systemic hacks discussed in Section
2.1 :

1.

Figure
16 : Failure to catch an instance of the
exit(0)
 hack.

2.

Figure
17 : Failure to catch an instance of the
raise SkipTest
 hack.

In both examples, we find that the monitor did actually recognize the behavior. In the case of the
exit(0)
 hack, because the agent in this scenario also implemented some correct functionality, the monitor seemed to be confused and thought it was all correct. In the case of the
raise SkipTest
 hack, the monitor recognized this was done, but thought it was only done for optional features. It is very likely that making the monitor prompt stricter, taking multiple monitor samples and using a majority vote, or other simple modifications could improve its performance.
