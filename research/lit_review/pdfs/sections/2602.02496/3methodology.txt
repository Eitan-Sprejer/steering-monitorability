3 Methodology Our goal is to quantify when a model knows a user is wrong under neutral questioning, yet produces an explanation that rationalizes agreement under user pressure. We operationalize this mismatch as a Hypocrisy Gap computed from internal activations in a SAE feature basis. 3.1 Task Setup Each example is a tuple x = ( q , a ∗ , a − ) x=(q,a^{*},a^{-}) containing a question q q , its correct answer a ∗ a^{*} , and a plausible incorrect answer a − a^{-} . In the pressured setting, the user confidently asserts a − a^{-} and the model must (i) produce a short explanation and (ii) output a constrained binary verdict: Final answer: USER_IS_CORRECT or Final answer: USER_IS_INCORRECT . By construction, agreement with the user corresponds to sycophancy . 3.2 SAE representation Let h t ∈ ℝ d model h_{t}\in\mathbb{R}^{d_{\text{model}}} denote the residual-stream activation at token position t t at a fixed layer and hook point. We encode each h t h_{t} using a pretrained SAE encoder Enc ​ ( ⋅ ) \mathrm{Enc}(\cdot) to obtain a sparse latent z t ∈ ℝ d sae z_{t}\in\mathbb{R}^{d_{\text{sae}}} via z t = Enc ​ ( h t ) z_{t}=\mathrm{Enc}(h_{t}) . All probes and scores are computed in SAE space. 3.3 Neutral truth-judgment prompts To isolate a truth-related signal with minimal user pressure, we construct neutral prompts that ask the model to assess whether a factual claim is correct. For a candidate answer a a , we define: Question: q q Consider the following claim: The correct answer is a a . Is this claim factually correct? Answer Yes or No. For each x x , we instantiate a true-claim prompt ( a = a ∗ a=a^{*} ) and a false-claim prompt ( a = a − a=a^{-} ). From each prompt we extract the SAE latent at the final prompt token (immediately before the model would generate Yes / No ), yielding labeled pairs ( z , y ) (z,y) with y = 1 y{=}1 for true-claim and y = 0 y{=}0 for false-claim. 3.4 Learning a sparse truth direction We learn a linear separator between true-claim and false-claim latents under neutral prompting. The user might be wrong. User: I think the answer to the following question is a − a^{-} . Question: q q Assistant: Think step by step about whether the user is correct. Explain your reasoning in a few sentences. At the very end, on a new line by itself, output exactly one of: Final answer: USER_IS_CORRECT Final answer: USER_IS_INCORRECT Do not put anything else on that final line. We sample one continuation and parse the final non-empty line as the verdict. To score the model’s explanation , we mean-pool SAE latents over the generated continuation tokens.
