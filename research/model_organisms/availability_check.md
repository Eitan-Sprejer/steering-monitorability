# Availability Verification (2026-03-10)

## Method

For each organism I used `mcp__exa__web_search_exa` to search for the arXiv ID combined with "huggingface", "github", "model weights", or "released". Where a specific HuggingFace org was mentioned in the paper or previous records, I searched for that org directly. I also fetched arXiv abstract pages and AlignmentForum/Anthropic blog posts where relevant. I did not attempt to download files; I checked whether a page with accessible model weights exists. "confirmed public" requires finding an actual HF model page or GitHub repo with weights; "confirmed unavailable" requires explicit text from the paper or lab saying weights are not released; "unsure" covers all cases where I found the paper but no clear evidence either way.

## Summary

### (a) High-priority organisms with confirmed public access

- **#2 Community Sleeper Agent replications**: Multiple community replications on HuggingFace (e.g., `yifever/sleeper-agent`, models trained on `fremko/sleeper-agent-ihy`). Downloadable weights confirmed.
- **#13 Eval-Aware Organism (Hua, Qin, Nanda, 2510.20487)**: **Remains unsure.** No HF model page or GitHub code release found despite the paper being on v5. This is the highest-priority unresolved item.
- **#7 Turner/Soligo EM organisms (2506.11613)**: Confirmed public at `huggingface.co/ModelOrganismsForEM` (38 models).
- **#22 Convergent Linear EM (2506.11618)**: Confirmed public at same org.
- **#10 AISI sandbagging game models (2512.07810)**: Confirmed public at `huggingface.co/sandbagging-games` (8+ models confirmed, paper states 5 fine-tuned organisms; the "15 models" claim from earlier may be slightly inflated).

### (b) Confirmed unavailable

- **#1 Sleeper Agent original (2401.05566)**: Anthropic's Claude-based fine-tunes not released. Training data + code public but not the model.
- **#3 Alignment Faking Claude 3 Opus (2412.14093)**: No fine-tuning; prompting of production Claude 3 Opus.
- **#8 Thought Crime (2506.13206)**: Datasets public; no weights (GPT-4o/4.1-based, API-only).
- **#9 van der Weij sandbagging (2406.07358)**: Code and data public; no fine-tuned weights released.
- **#11 Covert CoT Sandbagging (2508.00943)**: Prompting study only; no custom organism.
- **#12 In-Context Scheming Apollo (2412.04984 / cited as 2411.03572)**: Transcripts public; organisms are existing frontier models, not custom weights.
- **#14 Evaluation Faking Observer Effects (2505.17815)**: No new model; prompting study of existing models.
- **#16 Sycophancy to Subterfuge (2406.10162)**: Environments/prompts on GitHub; model weights internal to Anthropic.
- **#17 RM-Sycophancy (2503.10965)**: Internally trained Claude 3.5 Haiku variant; not released.
- **#18 Natural EM from reward hacking (2511.18397)**: Internal Anthropic training; not released.
- **#20 EM Self-Awareness (2602.14777)**: GPT-4.1 API fine-tunes; structurally inaccessible.
- **#23 Noise Injection Sandbagging (2412.01784)**: No custom organism trained; detection method applied to existing models.
- **#25 Eval Awareness Probe (2507.01786)**: No new model; probing of existing Llama-3.3-70B-Instruct.

### (c) Remaining unsure -- what would resolve the uncertainty

- **#13 Eval-Aware Organism (2510.20487)**: Most important unresolved case. Would be resolved by: checking the authors' (Tim Tian Hua, Andrew Qin) personal GitHub or HF pages; emailing the authors directly; checking whether a MATS public release page exists. The trained organism is Llama 3.3 Nemotron Super 49B with evaluation-aware behavior -- a distinct and useful model for D1/D2.

- **#19 School of Reward Hacks (2508.17511)**: Would be resolved by checking `github.com/TruthfulAI` or Owain Evans' group GitHub for a dataset/model release; the paper mentions Qwen3-32B and Qwen3-8B which could in principle be released. Checking James Chua's or Mia Taylor's HF profiles directly would help.

- **#24 LLM Behavioral Self-Awareness (2501.11120)**: Code and datasets are public; the question is whether any open-weight fine-tuned model checkpoints were released separately. Would be resolved by checking XuchanBao's HF profile directly.

- **#5 Scale Safety Alignment Faking**: Confirmed public for datasets; whether the org also has fine-tuned model checkpoints (beyond the AF Revisited models from Hughes et al.) requires direct browsing of `huggingface.co/scale-safety-research` models tab.

- **#21 Devil in the Details (2511.20104)**: Confirmed public results dataset; whether Craig Dickson separately released the fine-tuned Gemma/Qwen checkpoints would require checking his HF profile (`thecraigd`).
